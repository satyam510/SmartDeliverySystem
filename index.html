PEAS Description
Performance Measure

Minimize total path cost, delivery time, and nodes explored. Maximize successful delivery rate and route optimality. Penalize traversing high-cost terrain (traffic/mud).

Environment

Grid-based city map with multiple terrain types: normal roads (cost 1), traffic zones (cost 3), muddy/wet areas (cost 5), and impassable obstacles. Fully observable, deterministic, static during pathfinding.

Actuators

Movement motors (UP / DOWN / LEFT / RIGHT). Delivery arm/compartment. Status lights and display.

Sensors

GPS / position sensor. Terrain scanner (LiDAR). Obstacle detector. Odometer. Delivery confirmation sensor.

ENVIRONMENT PROPERTIES
• Fully Observable — complete grid known
• Deterministic — fixed move outcomes
• Sequential — actions affect future states
• Static — grid unchanged during search
• Discrete — finite grid cells
• Single-Agent — one delivery robot
A* Algorithm Theory
Evaluation function:
f(n) = g(n) + h(n)
g(n) — actual cost from start to n
h(n) — Manhattan distance heuristic
h(n) = |x₁−x₂| + |y₁−y₂|
TIME COMPLEXITY
O(b^d)
b = branching factor (≤4 for grid)
d = depth of optimal solution
With admissible heuristic: Optimal & Complete
A* vs DIJKSTRA
A* uses heuristic → explores fewer nodes
Dijkstra = A* with h(n) = 0 always
A* is typically 2–5× faster in practice
Both guarantee optimal path if h(n) admissible
HEURISTIC ADMISSIBILITY
Manhattan distance is admissible because it never overestimates the true cost when min terrain cost ≥ 1. This guarantees A* finds the globally optimal path.
DATA STRUCTURES
• Open List — Min-heap (Priority Queue)
• Closed Set — Hash set for O(1) lookup
• Parent Map — Dict for path reconstruction
• g-score Map — Best cost to each node
